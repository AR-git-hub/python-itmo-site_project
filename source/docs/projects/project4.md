# **Проект 4**

## Цель и описание работы

**Цель:** Построение оптимальной модели машинного обучения для задачи классификации (прогноз дефолта заемщика) на наборе данных с дисбалансом классов.  

**Описание:** В ходе работы проведена предобработка данных, EDA и тестирование  алгоритмов из `scikit-learn`. Основной упор сделан на поиск модели, способной эффективно работать в условиях нелинейности признаков и численного превосходства одного из классов.


### Современные тренды 

**Лидеры:** Random Forest, Gradient Boosting, Neural Network, Boosting Decision Tree и Voting

## Дополнительные исследования с помощью LLM

*Ответ сгенерирован LLM*

### 1. Почему 0.86 — это предел (Анализ)

- **Шум и перекрытие:** Классы сильно перемешаны. У заемщиков с одинаковыми анкетами бывают разные исходы, что создает «стеклянный потолок» точности.
- **Слабость простых моделей:** Линейные (SVM, LogReg) и метрические (kNN) модели не справляются с нелинейной структурой данных.
- **Преимущество бустинга:** `HistGradientBoosting` (0.86) победил нейронки (0.83) и лес (0.84), так как лучше всех находит сложные закономерности в таблицах.

### 2. Интеграция в Веб (FastAPI) — Пошаговый алгоритм

1. **Export:** Сохрани обученную модель и scaler через `joblib.dump()`.
2. **API:** Создай сервис на **FastAPI** (он быстрее Flask и Django).
3. **Loading:** Загрузи модель в память один раз при старте сервера.
4. **Endpoint:** Сделай POST-метод `/predict`, принимающий данные в JSON.
5. **Preprocessing:** Внутри API прогони входящие данные через сохраненный `scaler`.
6. **Response:** Верни клиенту вероятность дефолта через `model.predict_proba()`.

## Выводы

- Наивысшую точность показал **HistGradientBoostingClassifier (0.863)**. 
- **Граница точности (0.9):** Достижение порога 0.9 на текущем датасете невозможно из-за **высокой энтропии**.
- Для деплоя модели рекомендуется использовать **FastAPI**. Алгоритм включает: сериализацию (`joblib`), создание эндпоинта `/predict`, препроцессинг входящего JSON-пакета через сохраненный `scaler` и возврат вероятности дефолта.
- Согласно опыту соревнований **Kaggle**, для подобных задач стандартом являются градиентные бустинги (XGBoost, LightGBM), что подтверждено результатами данной работы.
