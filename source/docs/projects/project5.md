# **Проект 5**

## Цель и описание работы

**Цель:** Построение модели машинного обучения для прогнозирования рыночной стоимости недвижимости на основе набора характеристик.  
**Описание:** В ходе работы был выполнен полный цикл разработки: EDA, отбор наиболее значимых признаков, сравнительное тестирование различных архитектур.

****Преимущества Adam** — скорость, адаптивность, устойчивость к шуму**

- Адаптивный шаг для каждого параметра — каждый вес имеет свой индивидуальный learning rate.
- Быстрая сходимость — достигает минимума за меньшее число итераций.
- Устойчивость к шуму — хорошо работает на разреженных данных.


## Интеграция в веб-сервис

*Ответ сгенерирован LLM*

Вот алгоритм интеграции модели в веб-сервис (например, на **FastAPI**):

1. **Экспорт (Сериализация):** Сохраните обученную модель и скейлер в файлы `.pkl` с помощью библиотеки `joblib` или `pickle`.
2. **Инициализация:** Создайте веб-приложение и загрузите модель в оперативную память один раз при старте сервера (обработчик `@app.on_event("startup")`).
3. **Создание эндпоинта:** Опишите метод **POST** (например, `/predict`), который будет принимать входные данные в формате JSON.
4. **Предобработка:** Внутри метода преобразуйте полученный JSON в `DataFrame` и прогоните через сохраненный ранее скейлер.
5. **Предсказание:** Вызовите метод `model.predict()` и верните результат пользователю в формате JSON.
6. **Контейнеризация:** Упакуйте сервис в **Docker-образ** для стабильного развертывания на любом сервере.

## Выводы и результаты

В ходе работы были протестированы 3 архитектуры: 
- случайный лес
- нейросеть (MLP) 
- градиентный бустинг

Удаление малозначимых признаков снизило точность, но позволило упростить модель без потери ключевой информации. 

Нейронная сеть на оптимизаторе Adam даже с предварительным масштабированием данных показала результат хуже бустинга (76к против 69к MAE), что подтверждает преимущество random forest при работе с табличными данными.

Лучшей моделью стал **HistGradientBoosting (MAE: 69 052)**. Он показал самый низкий уровень ошибок и наиболее стабильный результат на данных с большим разбросом цен.
